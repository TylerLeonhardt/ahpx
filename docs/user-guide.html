What is ahpx? #

AHP (Agent Host Protocol) is a WebSocket-based JSON-RPC protocol for managing AI agent sessions. It defines how clients connect to servers, create sessions, send prompts, receive streaming responses, and handle tool calls — all over a single persistent connection.

ahpx is the first full-featured AHP client. It's two things in one package:

  • A CLI for humans — connect to servers, manage sessions, send prompts, and stream responses directly from your terminal.
  • An SDK for automation — import AhpClient, ConnectionPool, FleetManager, and friends into your TypeScript/Node.js code and build on top of AHP programmatically.

Built by George's engineering team as the dispatch backbone for an autonomous CTO bot. The codebase is ~9,800 lines of TypeScript: ~6,700 application code and ~3,100 vendored protocol types.

Quick Start #

Get from zero to your first agent interaction in three steps.

Install

npm install -g ahpx

Add a server

ahpx server add local --url ws://localhost:8080 --default

Start prompting

ahpx exec --approve-all "What can you do?"

Starting an AHP Server

You need an AHP-compatible server to connect to. For example, a VS Code Server with the AHP endpoint enabled:

# Start a VS Code Server with AHP endpoint
./scripts/code-server.sh --agent-host-port 8080 --without-connection-token

Implicit Prompt Shorthand

Any text that isn't a recognized command is treated as a prompt — no subcommand needed:

# These are equivalent
ahpx exec "what files are in this directory?"
ahpx "what files are in this directory?"

# Pipe from stdin
echo "review this code" | ahpx

# With flags
ahpx --format json "summarize this repo"

SDK Reference #

Import ahpx into your TypeScript or Node.js code and build on top of AHP programmatically.

Connect and List Agents

import { AhpClient } from 'ahpx';

const client = new AhpClient();
const result = await client.connect('ws://localhost:8082');

console.log('Protocol version:', result.protocolVersion);
console.log('Agents:', result.agents);

await client.disconnect();

Create a Session

import { AhpClient } from 'ahpx';

const client = new AhpClient();
await client.connect('ws://localhost:8082');

const session = await client.openSession({
  provider: 'copilot',
  model: 'gpt-4o',
  workingDirectory: '/my/project',
  waitForReady: true,
  readyTimeout: 30_000,
});

console.log('Session URI:', session.uri);
console.log('Ready:', session.isReady);

Send Prompts

const result = await session.sendPrompt('Fix the failing test in auth.ts');

console.log('Response:', result.responseText);
console.log('Tool calls:', result.toolCalls);
console.log('State:', result.state); // "complete" | "cancelled" | "error"
console.log('Usage:', result.usage);

Multi-Session on One Connection

const client = new AhpClient();
await client.connect('ws://localhost:8082');

const session1 = await client.openSession({ provider: 'copilot' });
const session2 = await client.openSession({ provider: 'copilot' });

// Sessions are independent — events are pre-filtered by URI
const [result1, result2] = await Promise.all([
  session1.sendPrompt('Analyze the auth module'),
  session2.sendPrompt('Analyze the database module'),
]);

await session1.dispose();
await session2.dispose();
await client.disconnect();

Listen for Events

session.on('action', (envelope) => {
  const { action } = envelope;
  if (action.type === 'session/delta') {
    process.stdout.write(action.content);
  }
});

session.on('turnComplete', (turn) => {
  console.log('Turn finished:', turn);
});

session.on('error', (err) => {
  console.error('Session error:', err);
});

ConnectionPool

URL-keyed connection reuse — call getClient() multiple times with the same URL and you get the same client.

import { ConnectionPool } from 'ahpx';

const pool = new ConnectionPool();

// Returns existing client or creates new one
const client1 = await pool.getClient('ws://server-a:8082');
const client2 = await pool.getClient('ws://server-b:8082');
const client3 = await pool.getClient('ws://server-a:8082'); // reuses client1

console.log('Active connections:', pool.activeConnections); // 2

await pool.closeAll();

FleetManager

Health-aware, multi-server routing with pluggable strategies.

import { FleetManager } from 'ahpx';

const fleet = new FleetManager({
  connections: [
    { name: 'local', url: 'ws://localhost:8082' },
    { name: 'cloud', url: 'wss://cloud.example.com:8082', tags: ['gpu'] },
    { name: 'edge', url: 'wss://edge.example.com:8082', tags: ['fast'] },
  ],
  strategy: 'least-sessions',
});

// Select best server based on requirements
const server = await fleet.selectServer({ tag: 'gpu' });
console.log('Selected:', server.name, server.url);

// Get health of all servers
const health = await fleet.getHealth();
for (const h of health) {
  console.log(`${h.name}: ${h.status} (${h.latencyMs}ms, ${h.activeSessions} sessions)`);
}

Session Persistence

Resume sessions across process restarts.

import { SessionStore, SessionPersistence, AhpClient } from 'ahpx';

const store = new SessionStore();
const persistence = new SessionPersistence(store);

// Resume a previously saved session
const record = await store.get('session-id-123');
const client = new AhpClient();
await client.connect(record.serverUrl);

const outcome = await persistence.resume(record, client);
if (outcome.status === 'resumed') {
  console.log('Session resumed successfully');
} else if (outcome.status === 'not_found') {
  console.log('Session was disposed — create a new one');
}

// Sync local records with server
const sync = await persistence.sync(client, 'my-server');
console.log('Added:', sync.added, 'Removed:', sync.removed);

Auth Handling

import { AhpClient, AuthHandler } from 'ahpx';

const client = new AhpClient();
const auth = new AuthHandler(client, {
  token: process.env.AHPX_TOKEN,
  interactive: false,
});

client.on('notification', async (notification) => {
  if (notification.type === 'authRequired') {
    await auth.handleAuthRequired(notification.resource);
  }
});

await client.connect('wss://secure-server:8082');

Event Forwarding #

Stream agent events to external systems in real time via webhooks or WebSockets.

Webhook Forwarder

# Forward events to a webhook endpoint
ahpx exec --approve-all \
  --forward-webhook https://hooks.example.com/events \
  --forward-filter turn_complete,tool_call_complete \
  "fix the auth bug"

Features: batched delivery (configurable batch size and interval), retry with exponential backoff (up to 3 retries), event type filtering, and custom headers via --forward-headers.

WebSocket Forwarder

# Stream events over WebSocket in real-time
ahpx exec --approve-all \
  --forward-ws wss://dashboard.example.com/events \
  "deploy the new feature"

Features: real-time streaming, auto-reconnect with exponential backoff, backpressure handling (pauses at 1 MB buffer), and disconnect buffering (up to 10,000 events).

Multiple Forwarders

# Forward to multiple destinations simultaneously
ahpx exec --approve-all \
  --forward-webhook https://logs.example.com/ingest \
  --forward-ws wss://dashboard.example.com/live \
  --forward-filter delta,turn_complete,tool_call_start,tool_call_complete \
  "run the full test suite"

Architecture #

ahpx is structured in three layers, each adding higher-level abstractions on top of the one below.

CLI · bin.ts
Command parsing, output formatting, permission prompts, process lifecycle.
AhpClient
High-level typed API — sessions, state mirror, events, connection pool.
ProtocolLayer
JSON-RPC 2.0 message correlation — request/response pairing, notifications.
Transport · WebSocket
Raw WebSocket I/O — connect, send, close, events.

Supporting Systems

SessionHandle
Per-session wrapper with scoped events and prompt API.
ConnectionPool
URL-keyed connection reuse across sessions.
StateMirror
Client-side state tracking via pure reducers.
OutputFormatter
Strategy pattern: text, json, quiet.
PermissionHandler
Configurable tool approval policies.
EventForwarder
Webhook and WebSocket streaming.
FleetManager
Health-aware multi-server routing.
